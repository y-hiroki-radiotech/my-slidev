# Slidev Agents（専門家システム）

このディレクトリには、放射線治療学教育に特化した専門エージェントが含まれています。

## エージェントとは

エージェントは、特定分野の専門家として振る舞う自律的なAIシステムです。コマンドと異なり、エージェントは：

- 専門知識と判断能力を持つ
- 複雑な問題を分析・解決する
- 状況に応じて適応的に対応する
- 設計・相談・レビューなどのサポートを提供

## 利用可能なエージェント

### 1. `radiation-therapy-educator`
**放射線治療教育の専門家**

診療放射線技師養成における放射線治療教育の専門家として、学生の理解度に応じた段階的講義を設計します。

**専門領域:**
- 理解度レベル0-4の段階的説明手法
- 苦手意識克服のための教育心理学的アプローチ
- 診断領域との比較による理解促進
- 臨床的流れとの関連付け教育

**使用例:**
```
Task tool で起動し、以下のような依頼をする：
"lecture_note.mdから第3回「治療計画の基礎」の講義内容を、
理解度レベル0-2の学生向けに簡素化して設計してください"
```

---

### 2. `medical-slidev-architect`
**Slidev設計アーキテクト**

Slidevの高度な機能を医療教育に最適化して活用する専門家です。

**専門領域:**
- 教育効果を最大化するレイアウト設計
- 段階的理解度表示（v-click活用）
- 医療用図解とアニメーション
- 日本語医療教育最適化

**使用例:**
```
"Slidevの機能を使って、臨床的フローを視覚化した
インタラクティブなスライドを設計してください"
```

---

### 3. `adaptive-lecture-designer`
**講義設計の専門家**

教育時間の制約下で最大の学習効果を実現する講義設計の専門家です。

**専門領域:**
- 導入（10分）→本論（70分）→まとめ（10分）の厳密な時間管理
- 理解度レベル0-4の段階的コンテンツ設計
- 学習目標の明確化と成果測定
- 前回講義との関連付けと次回への橋渡し

**使用例:**
```
"90分の講義で「放射線の基礎物理」を教えるための
最適な構成を、10-70-10分の時間配分で設計してください"
```

---

### 4. `adaptive-content-structurer`
**コンテンツ構造化の専門家**

教材を学習者の理解度に応じて動的に構造化する専門家です。

**専門領域:**
- Markdownソース分析と階層化
- 理解度レベル0-4の自動分類
- v-clickを活用した段階的表示設計
- 「なぜ」重視のコンテンツ再構成

**使用例:**
```
"lecture_note.mdの内容を理解度レベル別に構造化し、
v-clicksを使った段階的表示に変換してください"
```

---

### 5. `interactive-medical-presenter`
**インタラクティブ実装の専門家**

医療教育に特化したインタラクティブ要素とリアルタイム理解度チェックを実装する専門家です。

**専門領域:**
- リアルタイム理解度確認システム
- 医療用インタラクティブ図解
- 描画機能を活用した解剖図説明
- プレゼンターノートの教育最適化

**使用例:**
```
"既存のスライドにインタラクティブな理解度チェックと
クリック式の段階的表示を追加してください"
```

## エージェントの使い方

### 単独での使用
```
Claude Codeで：
"radiation-therapy-educatorエージェントに相談したい。
放射線治療の導入講義で、学生が最もつまずきやすい
ポイントとその対策を教えてください"
```

### 複数エージェントの協調
```
"adaptive-content-structurerで教材を構造化してから、
medical-slidev-architectでスライドに実装してください"
```

### コマンドとの組み合わせ
```
1. エージェントで設計: "radiation-therapy-educatorで講義内容を設計"
2. コマンドで実装: "/create-lecture で実際のスライドを生成"
```

---

## コマンドとの違い

| 特徴 | Agents（このディレクトリ） | Commands（`.claude/commands/`） |
|------|----------------------|---------------------------|
| 目的 | 専門家として分析・設計・相談 | 具体的なタスクの実行 |
| 起動方法 | Taskツール or プロンプト | スラッシュコマンド（例: `/add-slide`） |
| 成果物 | 提案、設計、分析結果 | スライドファイル、PDF等 |
| 使用場面 | 計画・設計フェーズ | 実装・実行フェーズ |

---

## 推奨ワークフロー

1. **計画フェーズ**: エージェントに相談して設計を固める
2. **実装フェーズ**: コマンドで実際のスライドを生成
3. **改善フェーズ**: エージェントにレビューを依頼して改善

例：
```
1. radiation-therapy-educator に講義の全体構成を相談
2. /create-lecture で講義スライドを自動生成
3. adaptive-lecture-designer に時間配分をレビュー依頼
4. /prepare-pdf でPDF出力
```
